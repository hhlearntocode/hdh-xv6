1/ Which function called syscall?
Theo stack trace từ lệnh bt, hàm usertrap() đã gọi syscall().

2/ What is the value of p->trapframe->a7 and what does that value represent?
Lệnh: p /x *p cho thấy process hiện tại là initcode.
- Giá trị của p->trapframe->a7: 0x7
- Ý nghĩa: Thanh ghi a7 lưu system call number trong kernel/syscall.h, giá trị 0x7 tương ứng với SYS_exec. Nghĩa là tiến trình initcode đang thực hiện system call exec() để khởi chạy tiến trình init

3/ What was the previous mode that the CPU was in?
'p /x $sstatus': 0x200000022. Bit SPP = 0 nghĩa là CPU trước đó ở user mode, sau đó trap sang kernel.

4/ Write down the assembly instruction the kernel is panicing at. Which register corresponds to the variable num?
- Assembly instruction: lw a3,0(zero)
- Thanh ghi tương ứng với biến num là a3.
Lệnh lw a3, 0(zero) cố đọc dữ liệu tại địa chỉ 0x0 (NULL pointer), dẫn đến page fault.

5/ Why does the kernel crash? Is that confirmed by the value in scause above?
Kernel bị crash vì truy cập địa chỉ 0x0, vùng nhớ này không được ánh xạ trong không gian địa chỉ của kernel. Điều này tạo ra lỗi Load Page Fault, được xác nhận bởi giá trị scause = 0xd (tức 13 ở hệ thập phân), mã lỗi “Load Page Fault” trong kiến trúc RISC-V.

6/ What is the name of the binary that was running when the kernel paniced? What is its process id (pid)? 
- Tên binary: "initcode"
- Process ID (pid): 1
